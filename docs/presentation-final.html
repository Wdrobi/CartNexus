Problem & goals

Why CartNexus?

  • Retailers need both a public shop and back-office tools in one place.
  • Bilingual audience: Bangla + English across UI and content.
  • Operations teams need inventory, orders, and CMS without juggling many disconnected apps.

Project goals

  • End-to-end flows: browse → cart → checkout → order handling.
  • Role-based admin: catalog, inventory, CMS, subscribers, contact inbox.
  • Responsive UX and a maintainable React + REST architecture.

Solution overview

Single product: customer-facing storefront + secured admin, sharing one API and database.

Technology stack

  • Frontend: React 18, React Router, Vite, Tailwind-style utility CSS, i18next (EN/BN).
  • Backend: Node.js, Express, REST JSON APIs.
  • Data: PostgreSQL (products, orders, users, CMS, subscribers).
  • Auth: JWT for admin; customer auth for account area.
  • Content: Rich text editors for blog & legal pages.
  • Tooling: Git, environment-based config, production build pipeline.

Storefront

  • Home, categories, brands, product detail, cart, checkout, customer account basics.
  • Localized experience: language toggle, product copy in EN/BN where configured.
  • Footer & contact driven from store settings (addresses, chat links, newsletter).

Admin panel

  • Commerce: products, categories, brands, orders, inventory / stock views.
  • Content: home hero, blog, CMS for terms / privacy / FAQs.
  • Audience: newsletter subscribers, contact messages inbox.
  • Settings: store contact & chat URLs, theme-friendly admin UI (light/dark).
  • Discoverability: global search & activity-oriented notification feed.

Quality & security

  • Protected admin routes; API authorization for management endpoints.
  • Input validation and structured error handling on critical writes.
  • Responsive layout; accessibility-minded components where applicable.

Challenges & outcomes

  • Balancing rich admin UI with light mode readability and consistent tokens.
  • Keeping inventory and product models accurate for variants and stock.
  • Shipping a bilingual surface without duplicating business logic.

Outcome: a demonstrable full-stack product ready for academic review and further hardening.

Next steps
