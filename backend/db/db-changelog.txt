2026-04-09 — Home hero banner: table `home_hero` (single row id=1), public GET `/api/home/hero`, admin GET/PATCH `/api/admin/home-hero`; migration `db/migration_home_hero.sql`.
2026-04-09 — Home hero storefront: full-bleed image carousel with centered overlay copy, hover arrows, optional SQL refresh `db/migration_home_hero_cartnexus_banner_v2.sql`.
2026-04-09 — Admin product page fields: `GET/PATCH /api/admin/categories` includes `page_layout`; `GET /api/admin/products/:id` returns structured `description_sections_*` and `color_variants`; `POST/PATCH /api/admin/products` accepts `description_sections_en|bn` (JSON array) and `color_variants` (replace-all). No new DDL beyond existing `migration_product_page.sql` / `schema.sql`.
2026-04-09 — Checkout: tables `orders`, `order_items`; `POST /api/orders` (guest or optional `Bearer` customer token); `GET /api/orders` (customer JWT). Migration `db/migration_orders.sql`.
2026-04-09 — Admin dashboard analytics: `GET /api/admin/dashboard` (revenue, order counts, 7-day sales, top products, stock alerts, recent orders, category rollup, new customers 7d); `GET /api/admin/orders?limit=` (read-only list for admin UI).
2026-04-09 — Admin dashboard: `GET /api/admin/dashboard` supports optional `?revenueFrom=` & `?revenueTo=` (`YYYY-MM-DD`, max 400 days); response includes `revenueByDay` + `revenueRangeFrom`/`revenueRangeTo` (daily totals, excl. cancelled).
2026-04-09 — Admin dashboard: `?salesRange=7d|30d|90d` (default 7d) drives `dailySales` + `dailySalesSpan` + `salesRangeApplied`; adds `salesByZone` (from `orders.delivery_zone`), `retention90d`, `activityFeed`, `suggestedTasks`, `forecast` (heuristic from recent months).
2026-04-09 — Admin dashboard tasks: table `admin_task_completions` (migration `db/migration_admin_dashboard_tasks.sql`); `GET /api/admin/dashboard` includes `taskCompletions`; `PATCH /api/admin/dashboard/tasks` body `{ taskKey, done }`.
2026-04-09 — Realtime: HTTP server + `ws` WebSocket at `/ws/admin?token=<admin JWT>`; `broadcastDashboardRefresh` after successful `POST /api/orders`. Frontend connects for instant dashboard refetch.
2026-04-09 — `orders.delivery_zone`: `VARCHAR(64)` (was ENUM) via `db/migration_delivery_zone_varchar.sql`; `POST /api/orders` accepts any slug `^[a-z0-9_]{1,64}$` with known-zone fees from `DELIVERY_FEES` and default fee for new keys; admin dashboard zone bars/list/map driven by `salesByZone` (dynamic rows).
2026-04-09 — Admin inventory: table `inventory_stock_movements` (`db/migration_inventory_management.sql`); `GET/POST /api/admin/inventory` (list SKUs, summary, adjust stock, movement history).
2026-04-19 — Blog featured slot + navigation: column `blog_posts.is_featured` (`db/migration_blog_featured.sql`; also reflected in `migration_blog_posts.sql` / `schema.sql` for new installs). Public `GET /api/blog` sorts featured first; `GET /api/blog/:slug` adds `navPrevious`/`navNext` (by publish date). Admin blog form checkbox clears other featured rows when set.
2026-04-19 — CMS legal pages: table `cms_pages` (`db/migration_cms_pages.sql`) keys `terms`, `faqs`, `privacy` with `body_html_en|bn`. Public `GET /api/cms/pages/:key`; admin `GET /api/admin/cms-pages`, `GET/PATCH /api/admin/cms-pages/:key`. Storefront `/terms`, `/privacy`, `/faqs` render DB HTML when present (else existing static layout). Admin: Support section + `/admin/support/:pageKey`.
